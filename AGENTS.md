# Agent Notes

Before making project changes, read `README.md` and `docs/ROADMAP.md`. For deployment, service, sync, or Pi work, also read `docs/deployment.md`.

Keep these docs current:

- Update `README.md` when setup commands, scripts, repo layout, or current status changes.
- Update `docs/ROADMAP.md` when priorities, milestones, or open decisions change.
- Update `docs/deployment.md` when bootstrap, deploy, service, networking, or host assumptions change.
- Keep `SPEC.md` focused on product behavior and user-facing requirements.

Do not treat the fixture-backed management app as the final product. The intended system includes a management app, a Pi recorder app, and a minimal recorder emulator for testing.
